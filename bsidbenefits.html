in my last post i showed you how to setup the python version of the bot, and reasons you shouldn't use the BSID-JS. regardless, you won't believe it, but some did actually go down the rabbet hole of setting BSID-JS on cloudflare workers.

keep in mind, this is a hard thing to do even with the UI automated setup. regardless, i do have to acknowledge if you manage to get this behemoth of an administrative nightmare setup, there are some MASSIVE benefits to setting up the JS version on cloudflare workers, as this thing was meant for.

so, what are the benefits of getting bsid-js setup?

  1. Slash Commands & Diagnostics
    2. if you want your own selfhosted slash command setup, this is your goto. it has slash commands like /ping, and its ping command is a bit louder than bsid-py. it tells you exactly the worker execution time, gateway latency, the exact cloudflare colo datacenter, and even the Ray ID. it's nerding out, but useful nerding out.
  2. 24/7 Serverless Hosting (For Free!)
    3. cloudflare will keep your bot online almost all the time, even when you shut off your computer. i actually made bsid-js just because of this—operation cool off my PC. and it won't just keep it online, it'll keep it in several (i think it's over 300) data center locations across the globe. Plus, Cloudflare's free tier gives you 100,000 requests per day. unless you are running a mega-server, you will likely never pay a dime. compare that to your electricity bill for running a python bot 24/7.
  3. Smart Placement (Speed!)
    4. the worker will automatically be on cloudflare smart placement mode. this means cloudflare analyzes where your bot is talking to (Discord's API, Gemini's API) and automatically moves your bot's execution to a server physically closer to those APIs to cut down latency. it's literally faster than your home internet.
  4. More Security for Secrets
    5. cloudflare workers secrets API stores secrets a lot more securely than the typical .env file on your desktop. cloudflare takes your secrets and encrypts them at rest. because they are encrypted at rest, you won't have to worry about a hacker getting into your dashboard and simply reading your sensitive API keys, public key, and discord bot token.
  5. Auto Update (Set and Forget)
    6. provided you can use GitHub actions to store your secrets, the JS version now has a fully automated update function. every time i push an update to the code (new features, bug fixes), your bot can automatically rebuild and deploy itself. no more "git pull" and restarting scripts manually.
  6. User Install & Context Menus
    7. imagine having your own user application that you install on your discord account (not just a server). you can right-click any image anywhere on discord, go to **Apps**, and click **Describe Image**. Wala! a description is provided within your session instantly. no need to invite the bot to that server, no need to copy-paste image links. it just works, everywhere you go.
  7. OCR & Admin Tools
    8. unlike the python version, this one now supports /ocr to extract text directly from images (great for reading screenshots). it also has a super-secret (owner only) /logs command that lets you audit your cloudflare account logs directly from discord. you feel like a hacker just running it.

yes, it can. it absolutely can. it has userinstall, it has secure secrets. the only thing the public bot does not have is auto updates because devs version (public bot) is controlled by the dev (me) which means it is updated by the dev himself (me). i don't need auto update. the only problem outside of my control is if the bot gets verified, and then the bot will have heavy load on the discord API. if this is not what you want, you should learn to host your own version.

what do i even get out of hosting either on my own? i can just use the public version!

regardless of which you host, the one big thing you get is image descriptions for your community. i, as a blind person, created this for the blind community to use and build upon. of course, there are other benefits. for example, when you use the dev version of the bot, you are relying on my API keys, like gemini, and hoping i don't have shit happen on my google account, therefor, terminating my account. you're also gambling i don't look through public bot images, image hashes, descriptions, and so on. when you host your own, you have to use your own gemini API key. this means that i can't see your API queries. that also means the only persons credentials you're gambling is your own. unless, that is, cloudflare, discord, or gemini does something stupid, you likely don't have to worry. and you definitely don't have to worry about the public bot going rogue because i left an API key public somewhere. the last benefit i can go on about is the fact that, well, your bot is distributed between 300 colo datacenters across the globe. who doesn't want that? very hard to do a DDOS attack on cloudflare.
