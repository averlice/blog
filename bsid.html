in this tutorial, we will be setting up the blindsoft image describer on cloudflare workers. strap in, it's gonna be a fun ride!

requirements

you will need a few things before we can get this party started.
  1. a cloudflare account: you will need a cloudflare account (free is fine) for this to work. if you do not ahve one, you can create one here
  2. discord developer credentials
    3. you will need to get discord credentials. this includes: bot token, application ID, and application public key. you can get creds for discord on the discord developer portal
  3. gemini API key
    4. you must have a gemini API key, which is used to send images over to the gemini API. you can get one here
  4. cloudflare account ID and API token
    5. this is used to parse audit logs for the /logs command. to get this, go to your account, manage account, API tokens, and create one. permissions required:
    • Account > Audit Logs > Read
    • Account > Worker Scripts > Edit (for auto-deployment)
    note: under permission scope, select your specific account.
  5. bot owner IDs
    6. the bot requires you have at least 1 bot owner ID. to get these, open discord user settings > advanced > developer mode and toggle it on. then, you can copy in any ID of the user you right-click on. note: owner IDs are separated by a comma with no space.

deployment: let's go!

login to the cloudflare dashboard and select your account. note: if you have access to another account in which you want to deploy, you can also select that account, granted you have permissions to change and setup workers.
  1. go to add a worker and select "import from GitHub".
  2. if it asks for it, follow the instructions to setup a git integration using GitHub.
  3. copy the repository URL when it asks for the GitHub repository you want to deploy: https://github.com/averlice/bsid-js.
  4. follow the instructions, give it a name (name it whatever you want).
  5. it will ask you for your bot token, your gemini API key, cloudflare credentials, application ID, public key, and owner IDs. take those credentials you got and paist them into the fields that follow.
  6. Smart Placement: Note that this bot is optimized for Cloudflare's Smart Placement mode to keep it snappy!
  7. click that big beautiful deploy button!
  8. look for the deployment url that cloudflare will give you. it will be workername.account.workers.dev and in the discord developer portal, in your applications general information, paist it in as an interaction endpoints URL.

wahay, it should now be working! yay! now, go pop the champagne, go tell your parents, whatever you want!

automatic updates (the cool part)

the bot now supports automatic updates via GitHub Actions. this means every time you push code, it updates itself like magic.

  1. go to your GitHub repository, then go to **Settings** -> **Secrets and variables** -> **Actions**.
  2. add the following repository secrets:
    • DISCORD_TOKEN
    • DISCORD_APPLICATION_ID
    • DISCORD_PUBLIC_KEY
    • GEMINI_API_KEY
    • OWNER_IDS
    • CLOUDFLARE_ACCOUNT_ID
    • CLOUDFLARE_API_TOKEN

your bot should now be up to date and stay that way!

to play with the bot and add features on your own, do the following. we will assume you have git installed.
  1. clone your repository.
    2. git clone https://github.com/yourgithubusername/yourrepositoryname.git

yay! it's running.

a quick note about offline status

in most cases, discord will show it as offline. this is primarily because cloudflare workers is an "online when woken" bot, meaning it only comes online when it is told to. in most cases, you can just run /ping on the bot, and if it shows latency along with detailed node information and Ray IDs, you are in the clear. if it responds with either "the application did not respond" or "the application didn't respond in time", that usually indicates there is a problem.

commands included

  • /gemini - Chat with Gemini 3 Flash.
  • /describe - Detailed image description for accessibility.
  • /ocr - Extract text from images exactly as it appears.
  • /ping - Check latency, Ray IDs, and Cloudflare diagnostic info.
  • /logs - (Owner Only) View recent Cloudflare audit logs directly in Discord.
  • /say - (Owner Only) Send plaintext.
  • /esay - (Owner Only) Send JSON-formatted embeds.
